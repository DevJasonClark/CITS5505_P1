HTML

HyperText Markup Language (HTML) documents are of a tree-structure, comprised of nested elements, consisting of tags and content. Tags typically form pairs that encapsulate content, and may include attributes (surrounded with quaotations "") which further alter the display or interpretation of content (adding style, classes, hypertext links, sources, etc), and are nested between the < and > symbols. The second tag of any pair includes the / symbol to 'close' the tag.

Comments are made (text not read by the browser) with <!--Comment-->, with multiple lines simply requiring one closing --> at the end

HTML is designed for content delivery. 


<!--Example Element:-->
<tag attribute="value"> content </tag>
<!--"tag" denotes the type of element.
    "attribute" denotes an element of the content to alter.
    "value" is what the attribute should be changed to.
    "content" denotes any text that should be displayed.
    "/" denotes the closing tag, thus the end of the element.-->

<!--Example Document Structure:-->
<!DOCTYPE html> <!--!DOCTYPE instructs the browser on what version of HTML this document is.-->
<html lang="en-AU">
<!--"html" denotes the document structure beginning
    "lang" instructs the browser on the predominant language this document displays.-->
  <head> <!--"Head" contains metadata on the document (charset, external resources, etc).-->
    <title>CITS5505 Project 1</title> <!--The main title for the page; not displayed.-->
      ...
  </head> <!--Close the head element.-->
  <body> <!--The "body" contains the majority of content for the page.-->
    <h1 style="font-weight:bold">This is a Heading</h1>
    ... <!--The above "h1" heading has been made bold.-->
  </body> <!--Close the body element.-->
</html> <!--Close the html document.-->

Style

Within HTML element tags, an attribute that can be changed is the contents "style." This ranges from text and background colour to positioning and spacing around other elements, with a semicolon ; separating individual style elements. 


<!--Example:-->
<div style="color:darkblue; background-color:lightcoral; display:block; margin:auto; text-align:center; width:75%;">

<!--Broken Down:-->
<div <!--We open with a "div" which is a division of the document (a section).-->
  style=" <!--The attribute we are altering is its style.-->
  color: darkblue; <!--Sets the text colour to dark blue.-->
  background-color: lightcoral; <!--Sets the text colour to light coral.-->
  display: block; <!--We set its display type to block, which adds line breaks before and after it.-->
  margin: auto; <!--Set the margin to automatic, adding even space to the left and right of the element (effectively floating it in the middle).-->
  text-align: center; <!--All text within the div aligns to the center.-->
  width: 75%"> <!--The divs width is set to 75% of the width of whatever it is inside.-->
This is how content within the above div appears. The text is centered and dark blue, the background colour is light coral, and the block has a line break before and after it

Notice I didn't say style had to look good.

The above div is long and difficult to read. We could potentially write it as per the example, with each new element of style being placed on a new line (extra whitespace between attribute elements has no real meaning), but our document would soon become much longer and more difficult to read.
To remedy this we have two main options.

1) We can place all of our style elements in the <head> of our file - the portion saved for metadata - and refer to these styles by using classes, such as: 


/* Note that within the style tag, as well as in Cascading Style Sheets (see below),

<head>
  <link rel="stylesheet" href="project1.css">
</head>

/* Or we can define styles in <head>: */ 
<head>
<!--HTML comment-->
  <style>
/* style comment */
    .example { /* Custom classes begin with '.', and we use curly brackets to contain each style element. */
      color: darkblue;
      background-color: lightcoral;
      display: block;
      margin: auto;
      text-align: center;
      width: 75%;
    }
    h1, h2 { /* We can apply a style to more than one element, separating them with ',' */
      ...    /* When specifying a style for a pre-existing tag-type we do not need '.' */
  </style>
</head>
  ...
  <div class="example"> Content <div>

2) We can place our styles in a separate document altogether, potentially applying them to multiple HTML documents without the need to rewrite the styles multiple times.

These documents are known as Cascading Style Sheets (CSS), explained further below.

CSS

Cascading Style Sheets (CSS) allow for the collection of styles in one or more documents, which can then be applied to numerous HTML documents concurrently in their <head>.
CSS documents follow the same format as individual styles defined within the <style> element in <head>, and contain no 'executable' code.

CSS is designed for content styling. 


* { /* '*' applies to all elements in the document. */
  font-family: 'Space Grotesk', sans-serif;
  font-size: medium;
}
...
.article {
  margin: auto;
  max-width: 1000px;
  min-width: 300px;
  padding: 10px 5px; /* Some style attributes can have multiple arguments. */
}

Multiple CSS documents can be applied to an HTML document, and just like with multiple individual styles they are applied in a top-down format. When applying multiple styles, 'later' defined styles and style sheets overwrite pre-existing defined attributes.

JavaScript

With HTML designed for content delivery and CSS designed for content styling, we have a strong basis for designing web applications that look nice and convey content in an easily-written and readable format. But so far we have described 'static' performance. What if we want to add more to both style and functionality?

JavaScript is a scripting language, designed for the web, that allows us to run code without the need for compilation. Execution is handled by the user's browser, and allows developers to add more interactive functionality to a web page. Style can be changed by the press of a button; forms can determine whether a users input is valid before sending it off to the server; elements on the page can be animated and manipulated on the fly.

Just like with style, javascript can be implemented within a HTML document or in its own file, similarly linked in the <head> tag, within a <script> tag. Comments in JavaScript are made just like in CSS as '/* comment */', except for multiline comments where you can simply use '//' at the beginning of any comment to denote that the remaining line is such. 


/* As with CSS, we can link to an external JavaScript document: */
<head>
  <script src="demonstration.js"></script>
</head>

/* Or we can define the script in <head> directly using <script>: */
<head>
  <script>
    function exampleFunction () {
      alert("This function brings up an alert in the users window.");
    }
  </script>
</head>

/* We can then call the script (or invoke it) by referencing it as functionName() */
<button onclick="exampleFunction">Press Me</button>

A function is first telegraphed by the word 'function', followed by its name, followed by parentheses, and then finally the body of the function contained within curly brackets '{}'. Functions are called by using their name and brackets 'myFunction()', with any necessary parameters contained within the brackets. Functions can then change a value on an internal document (the HTML document, variables in the JavaScript file, an external file of data, etc), return a value (based on user input or variables stored internally), or perhaps perform some function in the background, neither returning or changing anything.

Below we have a very simple script that reads the values of the X and Y boxes and then sends an alert to the window that provides the sum of the two numbers. Try changing the numbers.

Forms are another powerful tool introduced by HTML but greatly empowered by JavaScript. One such example is using JavaScript to parse user input and determine its validity. Observe the below form:


HTML and CSS alone (with the help of Bootstrap) allow us to set required fields, parse simple data such as names and emails or to ensure certain checkboxes are ticked, but what if we want to determine whether someone's password is secure enough (or at least to our standard format)?
The above form invokes the "verifyForm" function when the submit button is pressed and all other fields are filled.
The password is then verified to:

  • Contain only letters a through z (uppercase allowed), numbers and underscores,
  • Have a length between 6 and 10,
  • Start with a letter.

If the user's input fits the criteria, the form is submitted. (well, not this form. An alert will appear, instead). 


function verifyForm () {
  let usrInput = document.getElementById("pword").value;
  let allowed = /^[A-Za-z]\w{5,9}/;
  // The above 'allowed' variable is a "regular expression," which is a series of characters that denote a search pattern.
  // ^[A-Za-z] implies that the first character (^) must be a letter, uppercase or lowercase ([A-Z] and [a-z]).
  // \w{5,9} implies that there must then be between 5 and 9 ({5,9}) other characters of the 'word' family (\w)
  // which includes letters, numbers and the underscore symbol (_).
  
  if(usrInput.match(allowed))
      alert(usrInput + " is valid!");
  else
      alert(usrInput + " is invalid! Try again.");
}

These are basic implementations of JavaScript but provide interesting ways to add simple functionality to a web page. JavaScript can even be made to 'listen' out for certain events. These events could be performed by the user (clicking a button or portion of the screen, typing, etc) or by a state change or loading of resources (the page is loaded for the first time or closed, an image source can't be found, etc).

AJAX

There may situations in which you want JavaScript to do something, but can't rely on an immediate response. JavaScript is "synchronou," meaning that it executes line after line, essentially in one direction. If a developer wishes to retrieve data from a server and display that information in their web page, they may not know if the request will succeed at all, let alone how long it will take. AJAX, which stands for Asynchronous JavaScript and XML allows for asynchronous operation of JavaScript; allowing a script to continue operating without needing to wait for a response.

The way this typically works, is that the response itself will invoke the remaining operation to be completed.
