Pre 1990s

  • 1963
  • Ted Nelson Coins the terms ‘hypertext’ and ‘hypermedia’ [1]

“Let me introduce the word "hypertext" to mean a body of written or pictorial material interconnected in such a complex way that it could not conveniently be presented or represented on paper. It may contain summaries, or maps of its contents and their interrelations; it may contain annotations, additions and footnotes from scholars who have examined it."

Ted Nelson, Complex information processing: a file structure for the complex, the changing and the indeterminate [1]
  • 1980
  • Tim Berners-Lee develops ENQUIRE (Hypertext database program; text displayed on a screen with embedded hyperlinks to other documents) [2]
  • 1983
  • Domain Name Service (DNS) is first invented, allowing for translation between IP addresses and domain names, vastly simplifying to process of connecting to other servers, if only for not needing to remember the IP Address itself. [3]
  • 1986
  • DNS becomes standardized for the web [3]
  • 1989
  • Tim Berners-Lee proposes the world wide web, utilizing HyperText Transfer Protocol (HTTP) to send and receive HTML files, interpretable by any browser. [2] [4]

Early 1990s

  • 1990
  • Tim Berners-Lee develops the browser “WorldWideWeb” for the NeXT computer system [5]
  • 1991
  • Tim Berners-Lee invents HyperText Markup Language (HTML), based on Standard Generalized Mark-up Language (SGML) which was already an internationally recognised standard for mark-up text. HTML contains a collection of elements, which consist of tags and content, and provide the semantics for how said content should be displayed. Elements are of the form <tag attribute=value> Content </tag> Whilst HTML would begin with many of the standard tags included in SGML (such as paragraph tags <p>, heading tags <h1> through <h6>, lists - both ordered <ol> and unordered <ul> - containing list items <li>) it would also include hypertext links with an HREF attribute allowing for text to act as buttons, loading further resources when selected. [7]
  • 1993
  • Dave Raggett drafts HTML+ [7]
  • National Center for Supercomputing Applications (NCSA) Mosaic releases, leading to an explosion of popularity in browsers for inclusion of images and text on the same page. Brought web browsers to the non-technical user. [7] [8]
  • 1994
  • World Wide Web Consortium (W3C) is founded by Tim Berners-Lee. [2]
“...a Web standards organization… develops interoperable technologies (specifications, guidelines, software, and tools) to lead the Web to its full potential.”
  • Tim Berners-Lee and several colleagues write an article titled “The World-Wide Web” which outlined the necessary technologies and standardized specifications to implement the world wide web. [9]
  • Marc Andreessen founds Netscape and releases Netscape Navigator [8]
  • HTML working group is founded by the Internet Engineering Task Force (IETF) to help standardize HTML. [7]
  • The HTML Working Group defines HTML 2.0 [7]

Late 1990s

  • 1995
  • HTML is further extended with new tags, with great discussion emerging on whether HTML should include styling such as text and background colour. [7]
  • Microsoft releases Internet Explorer. [7] [8]
  • Dave Raggett publishes a draft of HTML 3 (the full realisation of HTML+) and is met with difficulty in having it standardized due to the large number of new inclusions. Various browser developers decide to implement features of HTML 3 as they saw fit, despite HTML 3 not being fully published. [7]
  • Dave Raggett forms a small group of representatives from each browser company to help streamline the standardisation of HTML where the HTML working group is struggling. [7]
  • W3 begin discussing Cascading Style Sheets (CSS) as a solution for styling HTML pages, potentially with multiple, otherwise conflicting, documents. Prior to this, developers would implement their own style sheets with non-standardised formats. [10]
  • Brandan Eich, whilst working at Netscape as a programmer, releases “Mocha” (temporarily then referred to as “LiveScript”) which would later become known as JavaScript. JavaScript, named either for Netscape’s support of Java applets or possibly to piggyback on Java’s popularity, was designed as a more dynamic language allowing for interpretation of code at run-time, with no need for compilation. This made it perfect for interpreting and reacting to events in web applications. [11]
  • The Document Object Model (DOM) level ‘0’ is conceived from JavaScript’s interpretation of user events and the manipulation of HTML elements. DOM outlines the tree-like structure of HTML and XML documents whereby each element of the document is nested within other nodes (or objects). This allows for the document to be traversed semantically and manipulations made on the fly. [12]
  • The HTML working group is dissolved. [7]
  • 1996
  • W3 forms the HTML Editorial Review Board (ERB) to again, help with standardising HTML. [7]
  • Cascading Style Sheet (CSS) is released, providing a unified standard for structure style sheets, and allowing for ‘cascading’ styles to overwrite each other where necessary, allowing for more comprehensive styling. [10]
  • HTTP 1.0 is released. [4]
  • 1997
  • HTTP 1.1 is released, clearing up certain ambiguities in HTTP 1.0, which then goes on to be the standard for over 15 years (with revisions). [4]
  • Netscape allows the European Computer Manufacturers Association (ECMA) to handle language specification for JavaScript. [11]
  • HTML 3.2 is released as the cross-industry specification. [7]
  • 1998
  • Formerly referred to as “Cougar,” HTML 4.0 is released. [7]
  • DOM level “1” becomes a W3C standardised recommendation, despite numerous updates to the now termed “Legacy” DOM since its release. [12]
  • EXtensible Mark-up Language (XML) is released as a simplified version of SGML, utilising developer-defined tags to structure information. XML differs from HTML in that XML is designed to carry data, whilst HTML is designed to display it. [13] [14]
  • 1999
  • W3C releases their recommendation for HTML 4.01 [6]

2000s Onwards

  • 2000
  • W3C releases their recommendation for eXtensible HyperText Markup Language (XHTML); a hybrid of XML and HTML more strict than HTML by itself, relying on more comprehensive error handling and structure. Where HTML continues to display as best as it can in the presence of errors, XHTML requires does not. [15]
  • DOM Level “2” is released. [12]
  • 2004
  • Web Hypertext Application Technology Working Group is formed to develop HTML, DOM and Fetch standards for web development. [16] [17]
  • DOM Level ”3” is released. [12]
  • 2005
  • Jesse James Garrett releases a paper titled “Ajax: A New Approach to Web Applications” that outlines the implementation of “Asynchronous JavaScript + XML” (AJAX). Ajax described as more of a methodology than a technology (or perhaps a collection of technologies) as it outlines the implementation of DOM, HTML, XHTML, XML, CSS and many others to create an asynchronous computing experience, similar to in desktop applications. This allows, among many other things, for developers to send requests to other servers without needing to wait for a response (ie, asynchronous). [11] [18]
  • 2008
  • WHATWG releases their first public draft for HTML 5. [19]
  • 2012
  • HTML 5 becomes Living Standard. [17]
  • 2014
  • W3C adds HTML 5 to their recommendation. [17]
  • 2015
  • HTTP/2 is released. [4]
  • DOM Level “4” is released, with further specification being handled by WHATWG. [12] [16]
